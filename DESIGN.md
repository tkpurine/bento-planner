# 弁当プランナーアプリ 設計ドキュメント

## アプリ概要

弁当の作り置きレシピ・ヘルシオホットクックの献立を管理し、
選択したメニューから買い物リストを自動生成するアプリ。

---

## コアフィーチャー

### 1. レシピ管理
- レシピの登録・編集・削除
- カテゴリタグ（作り置き向き／ホットクック専用／冷凍可など）
- ホットクック使用時は調理モード・調理時間も記録
- 何人分かの分量設定

### 2. 献立プランニング
- 作りたいメニューを複数選択
- 選択メニューの分量を人数・倍率で調整

### 3. まとめ買いリスト
- 選択メニューの材料を自動集計・重複をまとめ
- 食材カテゴリ別に整理（野菜／肉・魚／調味料など）
- 手持ち食材を除外するチェック機能
- リストをコピー・共有できる

### スコープ外（初期リリース）
- 栄養計算
- SNSシェア機能
- 外部レシピサイトの自動インポート

## 技術スタック

| レイヤー | 採用技術 | 理由 |
|---|---|---|
| フロントエンド | React + TypeScript + Vite | 実績・エコシステム最強、スマホアプリ化時も資産流用 |
| UIコンポーネント | Tailwind CSS + shadcn/ui | モバイルファースト、使いやすいコンポーネント群 |
| PWA | Vite PWA Plugin | ホーム画面に追加、オフライン動作 |
| バックエンド | Hono（Node.js） | 超軽量API、TypeScriptネイティブ |
| DB（現在） | SQLite + Drizzle ORM | ファイル1つ、VPSで運用コストゼロ |
| DB（将来） | PostgreSQL or Supabase | SQLiteと文法ほぼ同じ、移行コスト低 |
| インフラ | Docker + Nginx | VPS何でも動く、HTTPS対応 |

---

## ロードマップ

### Phase 1 - VPS + PWA（現在）
- VPS上でDockerコンテナ起動
- スマホブラウザからアクセス
- PWAとしてホーム画面に追加（見た目はほぼアプリと同じ）
- データはSQLiteで管理

### Phase 2 - クラウド化
- SQLite → PostgreSQL or Supabase に移行
- 複数端末での同期・バックアップ対応

### Phase 3 - ネイティブアプリ化
- Capacitor でWebアプリをラップ
- App Store / Google Play に公開
- フロントのコードはほぼそのまま流用

---

## アーキテクチャ設計方針

- **フロント・バックエンド分離**：将来のCapacitorアプリ化に備え、最初からAPI分離構成にする
- **Drizzle ORM採用**：コードをほぼ変えずにSQLite → PostgreSQLへ切り替え可能
- **shadcn/ui採用**：モバイルで使いやすいUI部品（ボタン・チェックボックス・スライダー等）が揃っている
